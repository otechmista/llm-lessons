Limitations: Why This Isn't ChatGPT

LLM Lessons is not a product. It's a study project. Understanding what it can't do is just as important as understanding what it can.

What This Model Can't Do

  • Answer reliably about topics not in the training data
  • Handle long conversations (64-character context window is tiny)
  • Use emoji or non-English characters (not in the vocabulary)
  • Produce fluent, consistent text (the model is too small)
  • Understand the world in any meaningful sense
  • Call APIs, search the web, or remember past sessions

Why It Doesn't Sound Like ChatGPT

The gap isn't a bug — it's a scale difference:

FactorLLM LessonsChatGPT-like models
Parameters~tens of thousandsbillions
Training data~4,000 charactershundreds of billions of tokens
Training timeminutes, on a laptopweeks, on thousands of GPUs
Tokenscharacterssubword pieces

Instruction tuningnoneextensive

ChatGPT quality comes from an enormous amount of work at every level: data collection, filtering, model design, training infrastructure, instruction tuning (teaching the model to follow instructions), safety systems, and production-grade inference.

This project implements the core mechanics. Those mechanics are real — the same architecture, the same training idea, the same inference loop. But quality requires scale.

Why There Are No Hardcoded Answers

It might be tempting to make app/infer.py check "if the user asks about pizza, return this string." But that would defeat the purpose.

This is a model study. If the model's output is weak, the right response is to understand why and improve model-related things:

  • More diverse training examples
  • More training steps
  • Bigger model (more layers, wider embeddings)
  • Lower temperature for more consistent output
  • Better prompt format matching the training data

Hardcoding answers would hide the model's actual behavior. We want to see it honestly.

RAG vs Training: Reading Notes Is Not Learning New Weights

RAG means Retrieval-Augmented Generation. It lets a model look up relevant notes before answering. RAG does not retrain the model. It gives the model extra context at answer time.

question
  -> embedding
  -> vector search
  -> relevant documents
  -> prompt with documents
  -> generated answer

Beginner version: training changes what is inside the model. RAG is an open-book exam: the model reads a retrieved page before writing the answer.

Common mistakes: saying RAG "teaches" the model permanently, trusting retrieved text blindly, or retrieving too much irrelevant text and confusing the model.

docs = vector_search(embed(question))
answer = llm.generate(question + docs)

Real Production GPT Optimizations

Production GPT systems use the same core loop you learned here, but they add engineering tricks to make huge models train and run at real-world speed and cost.

OptimizationWhat it isProblem it solves
QuantizationStore weights with fewer bitsLower memory and faster inference
MoEMixture of Experts activates only some expert layersMore capacity without using every parameter every token
RoPERotary position embeddingsBetter position handling than simple learned positions
Speculative decodingA small model drafts tokens, a big model verifiesFaster generation
Distributed trainingSplit work across many GPUsTrain models too large for one machine
Inference optimizationBatching, KV cache, kernels, memory planningLower latency and cost

Visual analogy: a large pizza shop needs smaller boxes, parallel ovens, prep stations, shortcuts, and specialists. The recipe is still pizza, but the operation is industrial.

This section is intentionally light. These ideas matter, but the foundation remains the same: tokens, embeddings, attention, logits, loss, and autoregressive generation.

The Real Lesson

The architecture of GPT-style models is learnable. The mechanics — tokenization, embeddings, attention, loss, backpropagation, inference — are all visible in this small codebase.

What you build here is a foundation. Everything in ChatGPT, Claude, Gemini, and Llama is an extension of these same ideas, just at a larger scale.
