LLM Lessons for Young Builders

Uma aula ludica para criancas e adolescentes entenderem, construirem e testarem uma IA pequena de verdade.

A ideia da aula

Imagine que vamos montar uma cozinha-laboratorio chamada Slice Pizza. A nossa IA nao nasce sabendo conversar. Ela aprende lendo exemplos simples sobre uma pizzaria ficticia e praticando uma missao repetida muitas vezes:

Dado o texto anterior, prever o proximo caractere.

Essa frase parece pequena, mas e o motor de sistemas GPT. Neste projeto, tudo fica pequeno de proposito para uma pessoa conseguir abrir os arquivos, ler o codigo e enxergar cada etapa.

Regras da nossa oficina

  • O projeto e educativo, nao um produto final.
  • As respostas publicas do modelo devem ser em ingles.
  • O tema do modelo e a pizzaria ficticia Slice Pizza.
  • Quando a pergunta foge do tema, a resposta que queremos ensinar e: I do not know how to answer that.
  • O modelo deve aprender pelo checkpoint treinado, sem respostas coladas no codigo.

Mapa da jornada

dataset
  -> texto vira tokens
  -> tokens viram numeros
  -> numeros viram vetores
  -> posicao entra na mistura
  -> atencao olha para o passado
  -> blocos transformer refinam a informacao
  -> logits pontuam o proximo caractere
  -> loss mede o erro
  -> backpropagation ajusta pesos
  -> checkpoint guarda o aprendizado
  -> inferencia gera uma resposta
  -> API deixa outro programa conversar com a IA

Etapa 1: Preparar os ingredientes

Arquivo principal: app/dataset.txt

O dataset é o caderno de receitas da IA. Ele agora mistura blocos de conhecimento com conversas curtas em inglês, repetidas e organizadas com os mesmos rótulos usados na inferência:

Customer: What pizza do you recommend?
Assistant: I recommend the Margherita pizza.

Pense no dataset como as frases que uma turma ensina para um colega novo. Se as frases forem claras, o colega entende o padrao. Se forem confusas, ele tambem aprende confusao.

Atividade: abra o dataset e encontre uma pergunta sobre pizza e uma pergunta fora do tema.

Etapa 2: Transformar conversa em um texto unico

Arquivo principal: app/context.py

Uma interface de chat parece ter várias mensagens separadas, mas o modelo recebe uma única sequência de texto:

Customer: What pizza do you recommend?
Assistant:

O rótulo aberto Assistant: é como dizer: "agora continue daqui como assistente".

Atividade: explique para outra pessoa por que treino e inferencia precisam usar os mesmos marcadores.

Etapa 3: Quebrar texto em pecinhas

Arquivo principal: app/tokenizer.py

Computadores nao pensam em letras como nos. Entao o tokenizer transforma cada caractere em um numero:

"pizza" -> ["p", "i", "z", "z", "a"] -> [32, 17, 51, 51, 8]

Neste projeto, usamos tokenizer por caractere porque ele e facil de investigar. Modelos grandes usam pedaços de palavras, mas a ideia central e parecida.

Atividade: pense em uma palavra nova. Ela so pode ser codificada se todos os caracteres existirem no vocabulario.

Etapa 4: Criar desafios de treino

Arquivo principal: app/batch.py

O batch builder cria pequenos desafios. O alvo e sempre o texto deslocado uma posicao:

texto:   p i z z a
input:   p i z z
target:  i z z a

Em cada posicao, o modelo tenta adivinhar o proximo caractere.

Atividade: faca esse deslocamento com a palavra lab.

Etapa 5: Dar significado aos numeros

Arquivo principal: app/embedding.py

O numero 32 sozinho nao significa "p". Ele e so uma etiqueta. O embedding troca cada etiqueta por uma lista de numeros treinaveis, chamada vetor.

token id -> vetor aprendido

No comeco, esses vetores sao aleatórios. Durante o treino, eles mudam para ajudar o modelo a prever melhor.

Atividade: compare isso com transformar nomes de estudantes em crachas com informacoes úteis para uma atividade.

Etapa 6: Mostrar a ordem das pecinhas

Arquivo principal: app/position.py

Sem posicao, a frase pizza is good poderia parecer igual a good is pizza. O modelo precisa saber onde cada caractere esta.

vetor final = embedding do token + embedding da posicao

Atividade: escreva duas frases com as mesmas palavras em ordem diferente. O significado mudou?

Etapa 7: Ensinar a olhar para trás

Arquivo principal: app/attention.py

A self-attention permite que cada token olhe para tokens anteriores e escolha quais ajudam mais. A mascara causal impede que ele veja o futuro.

posicao atual pode ver: passado + presente
posicao atual nao pode ver: futuro

Isso mantem o treino honesto. Se o modelo visse a resposta antes de tentar, ele estaria colando.

Atividade: explique por que uma prova com gabarito aberto nao mede aprendizado.

Etapa 8: Repetir o raciocinio em blocos

Arquivos principais: app/feedforward.py, app/block.py, app/model.py

Um bloco transformer combina:

  • attention, para misturar contexto entre posicoes
  • feed-forward, para transformar a informacao de cada posicao
  • normalizacao, para manter os numeros estaveis
  • conexoes residuais, para preservar informacao importante

O model.py junta tudo: embeddings, posicoes, blocos transformer e camada final de previsao.

Atividade: desenhe o caminho de um token entrando no modelo e saindo como pontuacao para o proximo caractere.

Etapa 9: Ver as pontuacoes do proximo caractere

Arquivo principal: app/model.py

O modelo nao responde diretamente com uma letra. Ele produz logits, que sao pontuacoes para todos os caracteres possiveis do vocabulario.

proximo caractere:
  "a" -> 1.2
  "e" -> 0.4
  "p" -> 3.8

Depois, a etapa de sampling escolhe um token a partir dessas pontuacoes.

Atividade: se p tem a maior pontuacao, qual caractere voce esperaria que fosse escolhido em uma configuracao bem previsivel?

Etapa 10: Medir o erro

Arquivo principal: app/loss.py

Loss e a nota do erro. Se o modelo acerta com confianca, a loss fica menor. Se ele aposta errado, a loss aumenta.

previsao boa -> loss menor
previsao ruim -> loss maior

Atividade: durante o treino, observe se a loss tende a cair. Ela pode oscilar, e isso faz parte do processo.

Etapa 11: Aprender com o erro

Arquivo principal: app/train.py

O treino repete este ciclo:

pegar batch
rodar o modelo
calcular loss
fazer backpropagation
ajustar pesos
mostrar logs

Backpropagation e o mecanismo que descobre quais pesos precisam mudar para reduzir o erro.

Comando:

python app/train.py

Etapa 12: Guardar o aprendizado

Arquivo principal: app/checkpoint.py

Depois do treino, o checkpoint salva:

  • pesos do modelo
  • vocabulario
  • configuracao

Sem vocabulario compativel, os mesmos numeros poderiam significar caracteres diferentes. Por isso o checkpoint precisa guardar mais do que pesos.

Atividade: explique por que salvar apenas os pesos nao basta.

Etapa 13: Gerar uma resposta

Arquivos principais: app/infer.py, app/sampling.py

Na inferencia, o modelo gera um caractere por vez:

prompt -> modelo escolhe 1 caractere
texto cresce -> modelo escolhe outro caractere
repete ate terminar

Comando:

python app/infer.py --prompt "What pizza do you recommend?"

Para estudar o texto completo produzido pelo modelo:

python app/infer.py --raw-model --prompt "What pizza do you recommend?"

Etapa 14: Conversar por uma API

Arquivos principais: app/api.py, app/schemas.py

A API e uma porta HTTP. Ela nao deve conter logica do modelo. Ela recebe uma requisicao no formato de chat, valida os dados e delega para a inferencia.

Comando:

uvicorn app.api:app --reload

Depois, abra:

http://127.0.0.1:8000/docs

Etapa 15: Testar como cientista

Arquivos principais: app/test_*.py

Testes ajudam a investigar se as regras continuam verdadeiras:

  • a mascara causal bloqueia o futuro
  • a geracao e autoregressiva
  • o checkpoint restaura o modelo
  • prompt vazio e rejeitado
  • API segue o contrato esperado

Comando:

python -m pytest

Mini roteiro para uma oficina

  1. Leia o dataset em voz alta.
  2. Mostre os marcadores <|user|> e <|assistant|>.
  3. Transforme uma palavra em caracteres e IDs.
  4. Monte input e target deslocados.
  5. Explique embeddings como vetores que aprendem.
  6. Mostre que posicao muda significado.
  7. Explique attention como olhar para o contexto anterior.
  8. Rode o treino e observe a loss.
  9. Rode a inferencia e compare respostas.
  10. Rode os testes e veja o projeto se defender de erros.

Perguntas de pesquisa

  • O que acontece se o dataset tiver poucas frases?
  • O que muda se o block_size aumentar?
  • Por que temperatura alta pode deixar a resposta estranha?
  • Por que uma IA pequena pode memorizar em vez de generalizar?
  • Como o modelo aprende a responder I do not know how to answer that. sem regra colada no codigo?

Fechamento

O LLM Lessons mostra que uma IA conversacional nao e magia. Ela e uma sequencia de transformacoes: texto, tokens, vetores, atencao, previsao, erro, ajuste e repeticao.

Quando voce entende cada etapa, deixa de apenas usar IA e comeca a investigar como ela funciona.
