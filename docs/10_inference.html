Inference: The Model Answers Your Question

Training is over. Weights are saved. Now you want to actually use the model.

Inference is the process of using a trained model to generate text. No more learning happens here — just predicting, one character at a time.

Where This Lives

app/infer.py

How to Run It

Ask a pizza question:

python app/infer.py --prompt "What pizza do you recommend?"

Ask something outside the training domain:

python app/infer.py --prompt "What is the capital of France?"

See the full raw generated conversation:

python app/infer.py --raw-model --prompt "What pizza do you recommend?"

The Autoregressive Loop

The model generates text one character at a time. Here's the full loop:

  1. Take your prompt: "What pizza do you recommend?"
  2. Format it as a conversation with markers:

``txt <|user|> What pizza do you recommend? <|assistant|> ``

  1. Encode to token IDs
  2. Feed into the model → get logits (scores for each possible next character)
  3. Pick one character based on those scores (sampling)
  4. Add it to the sequence
  5. Go back to step 4 with the longer sequence
  6. Stop when the model generates <|end|> or we hit the max token limit

This is called autoregressive generation because each step feeds on the output of the previous one.

Two Output Modes

Normal mode: returns only the assistant's answer:

I recommend the Margherita pizza.

Raw mode (--raw-model): returns the entire generated conversation transcript:

<|user|>
What pizza do you recommend?
<|assistant|>
I recommend the Margherita pizza.
<|end|>

Both modes use the same model. The difference is just which part of the output gets shown.

Decoding: Greedy, Temperature, Top-K, and Top-P

The model does not directly write a sentence. It produces logits for the next token, softmax turns them into probabilities, and a decoding strategy chooses one token.

StrategyBeginner meaningResult
GreedyAlways pick the highest probability tokenPredictable, sometimes repetitive
TemperatureMake the probability curve sharper or flatterLow = careful, high = wild
Top-KOnly sample from the K best tokensRandom, but within a short list
Top-PSample from enough tokens to cover probability PAdaptive shortlist
logits = logits / temperature
logits = keep_top_k(logits, k=5)
probs = softmax(logits)
next_token = sample(probs)

Why outputs change: the same prompt can produce different continuations if sampling allows multiple likely tokens to compete. Decoding changes the answer style without retraining the model.

Common Beginner Mistakes

  • Thinking high temperature means smarter. It often means more chaotic.
  • Thinking top-k and top-p are the same. Top-k keeps a fixed count; top-p keeps a probability mass.
  • Forgetting that greedy decoding can get stuck in repetitive patterns.

KV Cache During Generation

Autoregressive generation repeats the model many times. Without a cache, each step rereads the whole context and recomputes attention notes for tokens it has already seen. A KV cache stores previous Keys and Values so the next step can reuse them.

without cache: read [prompt + old tokens] again every step
with cache:    reuse old keys/values, compute only the new token
cache.append(k_new, v_new)
next_token = model.step(q_new, cache)

Real GPT use: KV cache is one of the reasons chatbots can stream text quickly. It does not change what the model learned; it changes how much computation is repeated.

Why the Output Might Be Weird

This model is tiny. It was trained for a few minutes on a small pizzeria dataset. If it generates strange text, that's expected and educational.

Things to inspect if the output is poor:

  • How long did training run? (Check max_steps in config.py)
  • What was the final loss?
  • Is the checkpoint fresh?
  • Is the temperature setting high (more random) or low (more predictable)?
  • Does the dataset have enough examples of the kind of question you're asking?

Fixing these things is how you improve an LLM — not by hardcoding answers.
